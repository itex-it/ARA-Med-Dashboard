## Status Bar Design Contract

**Layout:** Horizontal flex row, `px-6 py-2`, `border-b`, white background. Left-to-right segments separated by `<Separator orientation="vertical" className="h-4" />`.

**Segments (left to right):**

1. **ARA-Med Status** (STATUS-01)
   - Indicator: 8px circle dot with color (green/yellow/red)
   - Label: `text-sm font-medium` — "ARA-MED aktiv" / "ARA-MED pausiert" / "ARA-MED gestört"
   - Dot animation: `animate-pulse` only when status is "aktiv"

2. **Praxis-Status** (STATUS-02)
   - Label only (no dot): `text-sm` — "Praxis geöffnet" / "Praxis geschlossen" / "Sondermodus"
   - Color: green/muted-foreground/yellow

3. **Aktiver Modus** (STATUS-03)
   - Label: `text-xs text-muted-foreground` — "Normalbetrieb" / "Urlaubsmodus" / "Vertretungsmodus" / "Überlastung"

4. **Aktive Gespräche** (STATUS-04 — active call count, different from open task count)
   - Label: `text-xs text-muted-foreground` "Aktive Gespräche:"
   - Badge: `bg-primary text-primary-foreground text-xs font-semibold rounded-full px-2 py-0.5`
   - Shows 0 when no active calls (neutral muted badge)

5. **Offene Aufgaben** (STATUS-04 — existing OpenTaskCounter, extend not replace)
   - Existing component: `OpenTaskCounter` — keep exactly as-is
   - Extend layout.tsx to pass it as one flex item among the segments

6. **ARA-Med Toggle** (STATUS-05) — right-aligned via `ml-auto`
   - `<Switch>` component with label: "ARA-MED" left of switch
   - Checked = aktiv, unchecked = pausiert
   - Permission gate: render only when user role permits toggle action
   - On toggle: optimistic UI update + Server Action call + Realtime confirmation
   - Loading state: `disabled` + spinner (Lucide `Loader2` with `animate-spin`)

**Client boundary:** The entire status-bar content strip (the `div.border-b` inner content) becomes a Client Component `<StatusBar tenantId={tenantId} />`. layout.tsx remains a Server Component and passes `tenantId` as prop. This mirrors the existing `OpenTaskCounter` pattern.

## Call Log Page Design Contract

**Route:** `/telefonate` (new page under `(dashboard)`)

**Page structure:**
```
<main>                               ← layout.tsx flex-1 p-6
  <div class="space-y-4">
    <div class="flex items-center justify-between">
      <h1>Telefonate</h1>            ← text-xl font-semibold
      <span>Live-Indikator</span>    ← text-xs text-muted-foreground "Echtzeit"
    </div>
    <CallLogTable tenantId={...} />  ← Client Component
  </div>
</main>
```

**Table columns (CALL-02):**

| Column | Header | Width | Content |
|--------|--------|-------|---------|
| Zeit | "Zeit" | 120px | `DD.MM. HH:mm` in `text-xs font-mono` |
| Nummer | "Nummer" | 140px | Masked: `+43 660 *** **67` in `text-sm font-mono` |
| Dauer | "Dauer" | 80px | `1:23` or `—` in `text-sm` |
| Patient | "Patient" | 80px | Recognized badge: green dot "Ja" / muted "Nein" |
| Intent | "Intent" | 160px | Intent string or `—` in `text-sm` |
| Sprache | "Sprache" | 60px | `text-xs text-muted-foreground` e.g. "DE" |
| Status | "Status" | 120px | Badge (see color tokens above) |
| Aktion | "" | 40px | Chevron-right icon, `text-muted-foreground` |

Table behavior:
- Full-width table: `w-full`
- `<thead>`: sticky, `sticky top-0 bg-background z-10`
- Row hover: `hover:bg-muted/50 cursor-pointer`
- Row click: opens Call Detail Sheet (full row is clickable)
- Live rows: new rows prepend at top with `animate-in slide-in-from-top-2 duration-300`
- Active (in-progress) call row: left border `border-l-2 border-primary`
- Empty state: centered in table body, see Copywriting section

**Real-time behavior (REALTIME-01, REALTIME-02):**
- Subscribe to `call_log` table via `useRealtimeChannel` (existing hook)
- INSERT: prepend new row, apply slide-in animation
- UPDATE: update existing row in-place (no position change)
- Loading skeleton: 5 rows of `<Skeleton className="h-11 w-full" />` on initial load

## Call Detail Sheet Design Contract

**Trigger:** Click anywhere on a call table row.

**Component:** shadcn `<Sheet side="right" />` — 480px wide on desktop, full-screen on mobile.

**Sheet header:**
- Title: "Gesprächsdetails" (`text-xl font-semibold`)
- Subtitle: timestamp + masked number (`text-sm text-muted-foreground`)
- Close button: default Sheet close (X icon, top-right)

**Sheet sections (top to bottom):**

1. **Zusammenfassung** (CALL-07)
   - `summary_short` as `<p className="text-sm">` — shown for all users with view right
   - `summary_structured` rendered as labeled list: Intent, Aktionen, Offene Punkte, Rückruf nötig

2. **Kontaktdaten** (CALL-03)
   - Masked number always visible
   - Full number: revealed inline only if user has `call-detail` permission
   - Permission-gated: Server Component passes `hasCallDetail: boolean` prop

3. **Ausgeführte Aktionen** (CALL-04, CALL-10)
   - List of `call_actions` rows: icon + label + MEDSTAR API status chip
   - Status chips: "Erfolg" (green), "Fehler" (red), "Ausstehend" (muted)

4. **Audio-Aufzeichnung** (CALL-05)
   - Permission-gated: render only if user has `audio` right
   - `<audio controls src={presignedUrl} className="w-full" />`
   - URL fetched on sheet open via `/api/calls/[id]/audio` (15-min expiry, per STATE.md C7 decision)
   - Loading state: skeleton bar while URL is fetching
   - Not rendered if `audio_url` is null on the call log row

5. **Transkript** (CALL-06)
   - Permission-gated: render only if user has `transcript` right
   - Collapsible: `<details>` / shadcn Accordion pattern
   - Default: collapsed. Label: "Transkript anzeigen"
   - Content: `<pre className="text-xs whitespace-pre-wrap font-mono bg-muted p-4 rounded-md">`

6. **KI-Entscheidungsweg** (CALL-10)
   - Collapsible, collapsed by default
   - Label: "KI-Entscheidungsweg"
   - Content: structured JSON or labeled steps from `conversation_events` (read-only display)

7. **Interne Notizen** (CALL-08)
   - `<Textarea placeholder="Interne Notiz hinzufügen..." className="text-sm" rows={3} />`
   - Submit: `<Button size="sm">Notiz speichern</Button>`
   - Intent-Korrektur: `<Select>` with intent options below textarea
   - Label: "Intent-Korrektur (optional)"

8. **Feedback** (CALL-09)
   - Three `<Button variant="outline" size="sm">` options side by side:
     "Korrekt", "Falsch erkannt", "Training nötig"
   - Selected state: `variant="default"` (filled)
   - One selection at a time

**Sheet scroll:** Content area is scrollable (`overflow-y-auto`), header is sticky within sheet.

## Interaction States

### ARA-Med Toggle (STATUS-05)

| State | Visual |
|-------|--------|
| ARA aktiv | Switch checked, green dot pulsing, label "ARA-MED aktiv" |
| ARA pausiert | Switch unchecked, yellow dot static, label "ARA-MED pausiert" |
| Toggling (optimistic) | Switch disabled, `Loader2 animate-spin` next to label |
| Error | Switch reverts, toast: "Aktion fehlgeschlagen. Bitte erneut versuchen." |

### Call Row Live Update

| Event | Visual |
|-------|--------|
| New call arrives (INSERT) | Row slides in from top (`animate-in slide-in-from-top-2`) |
| Call completes (UPDATE) | Row status badge transitions in-place (no layout shift) |
| Active call in progress | Left border `border-l-2 border-primary` + pulsing dot in status badge |

### Audio Player

| State | Visual |
|-------|--------|
| URL loading | Skeleton `<div className="h-10 w-full rounded bg-muted animate-pulse" />` |
| Ready | Native `<audio controls>` element, full width |
| No permission | Not rendered (no placeholder shown — permission-gated at server) |
| No audio available | Not rendered (call log row has `audio_url: null`) |

## DSGVO / Permission Gates

These are visual contracts — implementation gating is server-side per STATE.md architecture decisions.

| Element | Permission required | Visual when no permission |
|---------|-------------------|--------------------------|
| Full phone number | `call-detail` | Not rendered — masked number shown instead |

| KI-Entscheidungsweg | `call-detail` | Section not rendered |

All permission checks run server-side in the Server Component that renders the Sheet. The Client Component receives only what it is allowed to display.

## Implementation Notes for Executor

1. **Extend, not replace:** `OpenTaskCounter` stays. The status-bar `div.border-b` inner content becomes `<StatusBar tenantId={tenantId} />` as a Client Component. `OpenTaskCounter` is one segment inside it.

2. **New route:** Create `src/app/(dashboard)/telefonate/page.tsx` as a Server Component. It reads `tenantId` from JWT and passes to `<CallLogTable tenantId={tenantId} hasCallDetail={...} />`.

3. **Audio URL route:** Create `src/app/api/calls/[id]/audio/route.ts` — generates a presigned URL with 15-min expiry via Supabase Storage. Never expose at page-load. Fetch only when Sheet opens (per STATE.md C7 decision).

4. **Realtime in table:** `CallLogTable` uses `useRealtimeChannel` on table `call_log`. Initial load: Server Action or direct Supabase query (tenant-scoped). Prepend new rows; update existing rows by `id`.

5. **Toggle Server Action:** Create `src/app/actions/toggle-ara-med.ts` — updates `tenants.active_features.ara_med_active`. Returns typed state. Optimistic UI via `useOptimistic` (React 19).

6. **Sheet is lazy:** `CallDetail` Sheet content fetches audio URL and `call_actions` only on open — not at page load.

7. **Table pagination:** MVP — no pagination. Show last 100 calls ordered by `created_at DESC`. Add "Ältere laden" button only if > 100 rows exist.
