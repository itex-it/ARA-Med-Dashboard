## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | call_actions SELECT | mitigate | RLS SELECT policy uses cached tenant_id subquery — cross-tenant reads return zero rows |
| T-03-02 | Tampering | call_actions INSERT/UPDATE/DELETE | mitigate | INSERT requires WITH CHECK tenant match; UPDATE/DELETE additionally role-gated to operator/ordination_admin |
| T-03-03 | Elevation of Privilege | RLS bare auth.jwt() per-row form | mitigate | All policies use the cached `(SELECT auth.jwt()->...)` subquery form (C1) |
| T-03-04 | Tampering | tenants status columns | accept | tenants has no RLS by design (D-21); writes occur only via service-role Server Actions which enforce tenant_id from JWT — covered in 03-02 |
</threat_model>
