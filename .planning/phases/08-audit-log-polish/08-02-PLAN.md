<interfaces>
logAuditEvent call pattern (add after primary DB operation succeeds, before return):

```typescript
import { logAuditEvent } from '@/lib/audit'

// Inside the action, after successful DB operation:
await logAuditEvent({
  tenantId,
  userId: user.id,
  action: 'ACTION_CONSTANT',
  objectType: 'object_type',
  objectId: String(record.id ?? ''),   // use record ID if available
  oldValue: null,                       // use old record if available via prior SELECT
  newValue: { /* relevant fields */ },  // use new/updated record
})
```

For DELETE actions: pass old record as oldValue, newValue as null.
For CREATE actions: oldValue as null, newValue as the inserted record.
For UPDATE actions: oldValue as prior record (fetch before update if not already available), newValue as updated record.

If a prior SELECT of the old record would require a significant code change (e.g., a function that only receives new values), pass oldValue: null rather than restructuring. The key requirement is action + objectType + objectId are always populated.
</interfaces>
