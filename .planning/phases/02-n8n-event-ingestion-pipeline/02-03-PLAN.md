    Create `src/lib/hooks/useRealtimeChannel.ts` — a `'use client'` module exporting a hook:
    `useRealtimeChannel<T>({ table, tenantId, events, onChange })` where `table` is the Postgres table name, `tenantId` scopes the channel, `events` is an array of `('INSERT'|'UPDATE'|'DELETE')`, and `onChange` is a callback receiving the `postgres_changes` payload.

    Inside a `useEffect` keyed on `[table, tenantId]`:
    - Create a browser client via `createClient()`.
    - Create the channel with a tenant-scoped name: `supabase.channel(table + ':' + tenantId)` (C3 — channel name carries the tenant so two tenants never share a channel).
    - For each event in `events`, register `.on('postgres_changes', { event, schema: 'public', table, filter: 'tenant_id=eq.' + tenantId }, (payload) => onChangeRef.current(payload))`. The `filter` is the server-side tenant guard — defense in depth alongside RLS (C3).
    - `.subscribe()`.
    - The `useEffect` RETURN must call `supabase.removeChannel(channel)` (C4 — without this, channels leak and duplicate events accumulate).
    - Stabilise `onChange` with a ref (`onChangeRef`) updated in a separate effect, so a changing callback identity does not tear down and rebuild the subscription on every render.
    - If `tenantId` is falsy, skip subscription (return early) — no channel for an unauthenticated/unscoped state.

    Type the payload generic so `payload.new` / `payload.old` are `T`. Add a file comment referencing C3 (tenant filter) and C4 (cleanup).
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Supabase Realtime → browser client | postgres_changes broadcasts row data; client must subscribe with a tenant filter |
| Client Component → anon-key Realtime | the browser uses the anon key; RLS + channel filter are the isolation guarantees |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Information Disclosure | Realtime channel without tenant filter leaks other tenants' call/inbox events (C3) | mitigate | Channel name carries tenantId AND every `postgres_changes` subscription sets `filter: 'tenant_id=eq.' + tenantId`; RLS on the table is the server-side backstop |
| T-02-11 | Denial of Service | Realtime subscription not cleaned up → memory leak + duplicate event handling (C4) | mitigate | useEffect return calls `supabase.removeChannel(channel)`; onChange held in a ref so the channel is not rebuilt every render |
| T-02-12 | Information Disclosure | service-role client used in a Client Component (C5) | mitigate | Realtime hooks use only `createClient()` (anon key); `server.ts` carries `import 'server-only'` so the service-role client cannot reach the browser bundle |
</threat_model>
