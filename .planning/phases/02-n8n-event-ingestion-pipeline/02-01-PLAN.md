From supabase/migrations/20260522000003_rls_policies.sql (the D-16 RLS template — 4 separate policies):
```sql
CREATE POLICY {table}_tenant_isolation_select ON public.{table}
  FOR SELECT TO authenticated
  USING ((SELECT auth.jwt()->'app_metadata'->>'tenant_id')::uuid = tenant_id);

CREATE POLICY {table}_tenant_isolation_insert ON public.{table}
  FOR INSERT TO authenticated
  WITH CHECK ((SELECT auth.jwt()->'app_metadata'->>'tenant_id')::uuid = tenant_id);

CREATE POLICY {table}_tenant_isolation_update ON public.{table}
  FOR UPDATE TO authenticated
  USING ((SELECT auth.jwt()->'app_metadata'->>'tenant_id')::uuid = tenant_id
    AND (SELECT auth.jwt()->'app_metadata'->>'ara_role') IN ('operator', 'ordination_admin'));

CREATE POLICY {table}_tenant_isolation_delete ON public.{table}
  FOR DELETE TO authenticated
  USING ((SELECT auth.jwt()->'app_metadata'->>'tenant_id')::uuid = tenant_id
    AND (SELECT auth.jwt()->'app_metadata'->>'ara_role') IN ('operator', 'ordination_admin'));
```

|----------|-------------|
| authenticated staff → call_log/inbox_items | dashboard users read call data; RLS must scope to their tenant |
| Realtime publication → subscribed clients | postgres_changes broadcasts row data; channel-level tenant filter required (enforced in 02-03) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | call_log cross-tenant read | mitigate | RLS SELECT policy with cached `(SELECT auth.jwt()->'app_metadata'->>'tenant_id')::uuid = tenant_id` — cross-tenant reads return zero rows (C1/C3) |
| T-02-02 | Elevation of Privilege | assistant/viewer mutating call_log | mitigate | UPDATE/DELETE RLS policies gate on `ara_role IN ('operator','ordination_admin')` |
| T-02-03 | Information Disclosure | Realtime publication leaks rows to wrong tenant channel | accept (here) | Publication membership is global; per-tenant filtering is enforced at subscription time in 02-03 (`postgres_changes` filter `tenant_id=eq.{id}`) — flagged as key link there |
| T-02-04 | Tampering | raw phone number persisted in call_log | mitigate | No raw phone column — only `phone_hash` (SHA-256); DSGVO Art. 9 / STATE.md checklist |
</threat_model>
