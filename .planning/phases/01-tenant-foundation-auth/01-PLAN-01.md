<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Next.js | Cookie-based auth; JWT validated server-side on every request |
| Server Component → Supabase | Service-role key used only for admin operations; anon key for user-scoped queries |
| proxy.ts → Protected Routes | Auth gate before any protected page loads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | `createBrowserClient` in server file | mitigate | `import 'server-only'` in server.ts; build fails if imported in client bundle |
| T-01-02 | Tampering | env var `SUPABASE_SERVICE_ROLE_KEY` leaked via NEXT_PUBLIC_ prefix | mitigate | .env.local.example documents correct naming; never prefix with NEXT_PUBLIC_ |
| T-01-03 | Information Disclosure | `getSession()` used instead of `getUser()` in proxy | mitigate | proxy.ts explicitly uses getUser() which re-validates with Auth server; getSession() is never imported |
| T-01-04 | Elevation of Privilege | AAL1 user accessing AAL2-required routes | mitigate | proxy.ts checks ara_role + aal claim and redirects to /auth/verify-totp |
| T-01-SC | Tampering | npm package install | mitigate | All packages are well-known, published by Supabase/Vercel/shadcn; no unknown packages installed |
</threat_model>
