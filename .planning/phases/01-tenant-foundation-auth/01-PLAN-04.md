<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PATCH /api/settings/tenant | Writes tenant config; guarded by role check from JWT only |
| scripts/seed-tenant.ts | Service-role script; must never be deployed to the app; runs locally only |
| Vault secrets | Written via service role; never returned by any API endpoint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | PATCH tenant sends tenant_id in request body | mitigate | Route handler reads tenant_id exclusively from user.app_metadata.tenant_id (JWT); body tenant_id is ignored |
| T-04-02 | Information Disclosure | GET /api/settings/tenant returns Vault secret | mitigate | Route handler explicitly selects only config fields from tenants table; Vault is never queried in this route |
| T-04-03 | Elevation of Privilege | Assistant or Viewer calls PATCH tenant settings | mitigate | Route handler checks ara_role IN ('operator', 'ordination_admin'); returns 403 otherwise |
| T-04-04 | Information Disclosure | seed-tenant.ts committed with real keys in code | mitigate | Script reads keys from env vars only (process.env); keys are never hardcoded in source |
| T-04-05 | Denial of Service | Unrestricted PATCH calls exhaust DB write quota | accept | MVP single-tenant operation; rate limiting is a Phase 8 concern |
| T-04-SC | Tampering | supabase db push applies unexpected migration | mitigate | All 6 migrations are code-reviewed before push; blocking human checkpoint precedes this task |
</threat_model>
