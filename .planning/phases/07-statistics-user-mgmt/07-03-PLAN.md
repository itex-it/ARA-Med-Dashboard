Server Action: createUserAction(formData: FormData)
  Input (from formData):
    email: string (validated as email)
    role: AraRole ('assistant' | 'viewer' — ordination_admin cannot create same or higher role; operator can create any)
    permissions: string (JSON-stringified Partial<ModulePermissions>)
  Auth:
    5-step sequence; ALLOWED_ROLES = ['operator', 'ordination_admin']
    currentUserRole = user.app_metadata?.ara_role as AraRole
    currentUserPermissions: query user_tenant_roles for current user_id to get their permissions JSONB
  Delegation:
    if (!canGrantRole(currentUserRole, targetRole)) return error: 'Unzureichende Berechtigung'
    parsed target permissions: apply applyDelegationCeiling(currentUserPermissions, targetPermissions)
  Create:
    tempPassword: crypto.randomUUID().slice(0, 12).toUpperCase() + '!1'  (meets typical password rules)
    const { data: { user: newUser }, error } = await serviceClient.auth.admin.createUser({ email, password: tempPassword, email_confirm: true })
    if (error || !newUser) return { error: error?.message ?? 'Fehler beim Erstellen des Benutzers' }
    Insert user_tenant_roles: { user_id: newUser.id, tenant_id: tenantId, role: targetRole, permissions: ceiledPermissions }
    Return: { ok: true, tempPassword } — so admin sees it once

Server Action: updateUserAction(formData: FormData)
  Input (from formData):
    targetUserId: string (the user_tenant_roles.user_id to update)
    role: AraRole
    permissions: string (JSON-stringified Partial<ModulePermissions>)
    active: string ('true' | 'false')
  Auth:
    5-step sequence; ALLOWED_ROLES = ['operator', 'ordination_admin']
    Must verify targetUserId's current row belongs to same tenant_id (defense-in-depth)
  Delegation:
    Fetch current user's permissions from user_tenant_roles
    if (!canGrantRole(currentUserRole, targetRole)) return error
    apply delegation ceiling on permissions
  Update:
    serviceClient.from('user_tenant_roles').update({ role, permissions: ceiledPerms, active }).eq('user_id', targetUserId).eq('tenant_id', tenantId)
    If active changed to false OR role changed: serviceClient.auth.admin.signOut(targetUserId, 'global')
    revalidatePath('/benutzer')

Server Action: deactivateUserAction(formData: FormData)
  Input: targetUserId: string
  Auth: 5-step sequence; ALLOWED_ROLES = ['operator', 'ordination_admin']
  Prevent self-deactivation: if (targetUserId === user.id) return error: 'Eigenen Account nicht deaktivierbar'
  Update: serviceClient.from('user_tenant_roles').update({ active: false }).eq('user_id', targetUserId).eq('tenant_id', tenantId)
  Invalidate: serviceClient.auth.admin.signOut(targetUserId, 'global')
  revalidatePath('/benutzer')

Role selector:
  In InviteUserSheet/EditUserSheet, the role <select> only shows roles the current user CAN grant:
  - operator sees: operator, ordination_admin, assistant, viewer
  - ordination_admin sees: assistant, viewer (canGrantRole enforced)
  Implemented as: ROLE_HIERARCHY options.filter(r => canGrantRole(currentUserRole, r))
  Note: operator can also assign ordination_admin but NOT another operator (canGrantRole checks strictly lower level)
  Correction: canGrantRole(grantorRole, targetRole) = ROLE_HIERARCHY[grantorRole] > ROLE_HIERARCHY[targetRole]
  So: operator (3) > ordination_admin (2) > assistant (1) > viewer (0)
  operator can grant: ordination_admin, assistant, viewer (not operator — 3 > 3 is false)
  ordination_admin can grant: assistant, viewer (not operator or ordination_admin)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Server Actions | All user management input untrusted; Zod validated before use |
| Server Action → Supabase Admin API | Requires service-role key — only available server-side |
| Server Action → user_tenant_roles | Defense-in-depth: .eq('tenant_id', tenantId) on all DB ops even though service-role client is used |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
