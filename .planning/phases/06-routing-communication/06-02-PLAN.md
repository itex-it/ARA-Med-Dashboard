## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-R-01 | Spoofing | createRoutingRuleAction, updateRoutingRuleAction | mitigate | tenant_id from user.app_metadata only; never from formData — STATE.md C2 pitfall |
| T-06-R-02 | Tampering | deleteRoutingRuleAction, deleteVipNumberAction | mitigate | .eq('id', id).eq('tenant_id', tenantId) — both filters mandatory; service-role bypasses RLS |
| T-06-R-03 | Elevation | Routing mutations without role gate | mitigate | All 6 Server Actions check ara_role IN ('operator','ordination_admin') before any DB write |
| T-06-R-04 | Tampering | JSONB condition_value / action_value arbitrary content | mitigate | Zod validates structure of condition_value and action_value in createRoutingRuleAction/updateRoutingRuleAction before upsert |
| T-06-R-05 | Tampering | forward_to_number action with no number | mitigate | Zod .refine() in schema: if action_type='forward_to_number', action_value.number must be non-empty string; UI also shows inline error "Bitte geben Sie eine Weiterleitungsnummer ein." |
</threat_model>
