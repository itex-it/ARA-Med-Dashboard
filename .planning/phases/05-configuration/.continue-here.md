## Key Patterns Established in Phase 5

- `getAuthContext()` helper — shared auth+role check in every server action
- `createServiceRoleClient()` is **sync** (no await); `createServerClient()` is **async** (requires await)
- Every service-role DELETE/UPDATE: `.eq('id', id).eq('tenant_id', tenantId)` — both mandatory (service role bypasses RLS)
- Zod v4: `.issues[0]` not `.errors[0]`
- Supabase query results cast: `(raw as SomeRow[] | null) ?? []` (tables not in generated types)
- EU AI Act: `EU_AI_ACT_DISCLOSURE` is a **server-side constant**, never from client input
