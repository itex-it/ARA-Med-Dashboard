## Spacing Scale

Declared values — 8-point scale, multiples of 4:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge padding |
| sm | 8px | Compact field spacing, button gaps within a row |
| md | 16px | Default vertical spacing between fields; card inner padding |
| lg | 24px | Section padding (p-6 matches established layout pattern) |
| xl | 32px | Gap between configuration sub-sections |
| 2xl | 48px | Gap between major configuration module tabs |
| 3xl | 64px | Page-level vertical padding (not used — layout provides p-6) |

Exceptions:
- Touch targets for weekday checkboxes: minimum 44px height (accessibility — Austrian medical staff, potential mobile use)
- Time input fields: fixed width 80px to prevent layout shift across day rows
- EU AI Act disclosure lock banner: 8px top + 8px bottom internal padding enforced regardless of surrounding spacing

## Page Structure

Phase 5 lives at `/konfiguration`. It uses the existing dashboard shell (layout.tsx with StatusBar).

```
/konfiguration
  └── Tabs (horizontal, top-level navigation within the page):
        ├── Öffnungszeiten      (HOURS-01..03)

        ├── Begrüßungstexte     (TEXT-01..04)
        ├── Vertretung          (DEPUTY-01..04)
```

- Server Component page at `src/app/(dashboard)/konfiguration/page.tsx`
- Page passes `tenantId` from JWT as prop to each tab's Client Component
- Tab state via URL search param `?tab=oeffnungszeiten` (default: first tab)
- Each tab is its own Client Component file — no single monolithic config page component

## Module-by-Module Interaction Contracts

### Module 1 — Öffnungszeiten (HOURS-01..03)

**Layout:** Single card with three collapsible sections or three sequential sections with Separator between them.

**Section A — Wöchentliche Öffnungszeiten (HOURS-01)**

| Element | Spec |
|---------|------|
| Weekday row | 7 rows (Mo–So), each: `Checkbox` (44px touch) + day label (text-sm, 80px fixed width) + time range inputs (HH:MM, two `Input` fields 80px wide each) + "Geschlossen" toggle (`Switch`) |
| Day names | German abbreviations: Mo, Di, Mi, Do, Fr, Sa, So |

| Save feedback | Button shows `Loader2 animate-spin` during pending; success: no toast (silent persist); error: `text-xs text-destructive` below button |

**Section B — Sondertage und Ausnahmen (HOURS-02)**

| Element | Spec |
|---------|------|

### Module 2 — Terminarten (APPT-01..05)

**Layout:** Single `Table` with inline editing for synonyms. No Sheet.

| Column | Width | Content |
|--------|-------|---------|
| Terminart | flexible | Name from MEDSTAR (display-only text-sm) |
| Sichtbar | 80px | `Switch` — controls whether type appears in AI |
| KI-buchbar | 100px | `Switch` — disabled if Sichtbar is off |
| Nur intern | 100px | `Switch` — mutually exclusive with KI-buchbar |
| Synonyme | flexible | Comma-separated chip list + edit trigger |
| Standard | 80px | Radio (only one row can be active) — label "Standard" |
| PID=0 | 80px | `Switch` — only relevant for Führerscheinuntersuchung; tooltip explains rule |

**Synonym editing (APPT-03):**
- Click "Bearbeiten" icon (`Pencil` lucide, 16px) in the Synonyme cell
- Expands an inline sub-row (not a Sheet) below the appointment row
- Sub-row contains: `Textarea` (full width, min-h-16) with comma-separated synonyms, help text "Beispiel: Bauchweh, Magenschmerzen, Leibweh", `Button` "Synonyme speichern" size="sm"
- Only one sub-row open at a time — opening another collapses the previous

**APPT-04 — Standard-Terminart:**
- One row per table carries a `Badge` variant="default" label "Standard"
- Change via `Select` dropdown in a separate "Standardeinstellungen" section below the table
- Label: "Standard für unklare Beschwerden:" + `Select` listing all visible appointment types

**APPT-05 — PID=0-Regel:**
- Inline `Switch` in the table row for Führerscheinuntersuchung
- `Tooltip` on the Switch info icon: "Für Führerscheinuntersuchungen kann PID=0 genutzt werden. Der Patient muss nicht im MEDSTAR-System erfasst sein."

**Visual state rules:**
- If Sichtbar = off → entire row: `opacity-60`, KI-buchbar and Nur intern switches disabled
- If KI-buchbar = on → Nur intern switch disabled (and vice versa)
- Active Synonym editor row: `bg-accent/10 border-l-2 border-primary`

**APPT-01 data source note (for executor):**
- MEDSTAR appointment types are fetched server-side and passed as props — never fetched client-side
- Manual refresh `Button` variant="ghost" size="sm" icon `RefreshCw` label "Neu laden" triggers Server Action re-fetch

## Component Patterns

### Form Save Pattern (matches Phase 1 / Phase 4)

- Page: Server Component — reads JWT, passes `tenantId` as prop
- Form component: Client Component (`'use client'`) — `useActionState` (React 19)
- Server Action: validates `tenantId` from JWT (never from form body), runs Zod schema, upserts to Supabase
- Pending: `Loader2 className="h-3 w-3 animate-spin mr-1"` inside Button
- Error: `<p className="text-xs text-destructive mt-1">{state.error}</p>` below the CTA

### Section Header Pattern (locked — matches Phase 3/4)

```tsx
<p className="text-xs font-semibold uppercase tracking-wide text-muted-foreground mb-2">
  {sectionTitle}
</p>
```

### Inline Add Form Pattern

New entries (special days, FAQ, medications) use an inline form at the bottom of the table — not a Sheet or Dialog (Sheet is reserved for full detail views, Dialog for confirmations).

### Tab Navigation Pattern (top-level module tabs)

```tsx
<Tabs defaultValue="oeffnungszeiten" className="w-full">
  <TabsList className="mb-6">
    <TabsTrigger value="oeffnungszeiten">Öffnungszeiten</TabsTrigger>
    <TabsTrigger value="terminarten">Terminarten</TabsTrigger>
    <TabsTrigger value="begruessung">Begrüßungstexte</TabsTrigger>
    <TabsTrigger value="vertretung">Vertretung</TabsTrigger>
    <TabsTrigger value="medikamente">Medikamente</TabsTrigger>
  </TabsList>
  ...
</Tabs>
```

URL sync: tab state reflected in `?tab=` search param — prevents state loss on refresh.

### Switch Flag Pattern (appointment types, medications)

- `Switch` with `useOptimistic` + `useTransition` for instant toggle feedback — matches Phase 3 toggle pattern
- Spinner `Loader2` shown during pending
- Switch disabled during transition

## Accessibility Constraints

- Weekday checkboxes: 44px minimum click/touch target
- All `Switch` components: `aria-label` with descriptive German label (e.g. "KI-buchbar: Allgemeinuntersuchung")
- EU AI Act lock block: `role="region"` + `aria-label="EU-KI-Gesetz Art. 50 Pflichthinweis"` + no focusable children (read-only)
- Calendar popover: keyboard-navigable (shadcn Calendar component provides this)
- Confirmation dialogs: focus trap inside Dialog (shadcn Dialog provides this via Radix)
- Error messages: `role="alert"` on error paragraphs
